Data Flow
1. Bill arrives in Google Drive
A utility bill PDF is dropped into the designated Drive folder. All bill types (electricity, natural gas, water) go into the same folder. The system identifies the type and provider automatically.
2. Apps Script detects the new file
A time-based trigger runs every 15 minutes. It compares files in the Drive folder against a log of already-processed file IDs. New files are queued for processing.
3. Provider check against known registry
The system checks the detected utility provider against a registry of known providers (providers.json). If the provider is new and unrecognized, a special "New Provider Detected" alert fires for manual review before any data is written.
4. Claude API extracts all data fields
The full PDF is sent to the Claude API (primary extractor). Claude reads all pages and returns structured JSON: bill type, property ID, dates, usage, cost, estimation flag, demand, and demand cost.
5. Gemini API independently validates
The same PDF is sent to Google's Gemini API with identical instructions but without Claude's output. Gemini reaches its own conclusions independently, catching misreads before they reach the sheet.
6. Outputs are compared and validated
The system compares both outputs field by field. Numeric values must agree within 2% (to handle rounding differences). Dates must match exactly. Sanity checks run on all required fields regardless of agreement.
7a. Agreement → written to Sheet
If both APIs agree and all sanity checks pass, the data is written to the correct tab (Electricity, Natural Gas, or Water) with a sequential row number. The Audit Log is updated.
7b. Disagreement → Review Queue + email alert
If the APIs disagree on any field, or sanity checks fail, the bill goes to the Review Queue tab. An email alert fires immediately with the row number, filename, which fields disagreed, and both extracted values side by side.
8. Dashboard updated
After each run, the system writes an updated bills.json to this GitHub repository. The dashboard reads it and refreshes automatically every 60 seconds.
New Provider Detection

The system maintains a registry of known utility providers in data/providers.json. When a bill arrives from a provider not in the registry:

  1. The system sends a New Provider Detected email with the full extraction result from Claude.
  2. Verify the extracted values manually against the actual bill.
  3. If correct, add the provider name to the known array in data/providers.json in the GitHub repo.
  4. On the next run, the system will trust that provider and process its bills automatically.
You never need to write new parsing rules
The AI handles layout variation across providers naturally. The registry just tracks which providers have been verified at least once. After that first confirmation, all future bills from that provider process automatically.

When a Bill Is Flagged
You will receive an email alert
Subject: [Bill Parser] Review Required - Row #X - Electricity
The email includes the row number, filename, which fields disagreed, and both extracted values side by side.
  1. Open the email. Note the row number and which fields are in conflict.
  2. Open the original bill using the Drive link in the email.
  3. Look up the correct value directly from the bill.
  4. Open the Google Sheet → Review Queue tab. Find the row by number.
  5. Manually enter correct values into the appropriate utility tab. Mark the Review Queue row as RESOLVED.
The system never writes wrong data
A bill in Review Queue is always better than a wrong number in the sheet. The queue is the intended safety net, not a failure state.
Security

All sensitive credentials are stored in Google Apps Script Script Properties — Google's built-in encrypted key-value store. No keys are written in code or committed to GitHub.

Keys stored in Script Properties
  • CLAUDE_API_KEYAnthropic API key for primary extraction
  • GEMINI_API_KEYGoogle Gemini API key for validation
  • GITHUB_TOKENFine-grained PAT, repo contents:write only
  • GITHUB_REPOkeyona-rerev/TESTER_Resource-Bill-Parser-Github
  • SHEET_IDGoogle Sheet ID from the URL
  • DRIVE_FOLDER_IDDrive folder ID where bills are dropped
  • ALERT_EMAILEmail address for all system alerts

To rotate any key: Apps Script editor → Project Settings → Script Properties → edit the value. No code changes needed.

Rotating API Keys
  1. Generate the new key from the provider (Anthropic Console, Google AI Studio, or GitHub Settings).
  2. Open Apps Script editor → Project Settings → Script Properties.
  3. Find the relevant key, click Edit, paste the new value, save.
  4. Run diagnosConfig() from the editor to confirm it shows SET.
  5. Revoke the old key from the provider's dashboard.
Glossary
kWh (kilowatt-hour)
Unit of electricity consumption. Usage column for electricity bills is in kWh (unadjusted, pre-loss-factor).
m³ (cubic metres)
Unit of volume. Usage for natural gas and water bills is in m³.
Demand (kW)
Peak power draw during a billing period. Electricity only. Uses the unadjusted Demand kW figure.
Demand Cost
Charge for peak demand: Distribution + Transmission Network + Transmission Connection - Transformer Allowance (if any).
Ontario Electricity Rebate
Provincial credit on electricity bills. Appears as "CR" and is subtracted when calculating final cost.
This Cost You (Enbridge)
The natural gas cost field on page 1 of Enbridge bills. Already includes HST. Taken as-is.
Estimation (Yes/No)
Whether the meter reading was estimated by the utility (Yes) or based on an actual read (No).
Review Queue
Sheet tab where bills go when the two AIs disagree or sanity checks fail. No data is written to utility tabs until manually resolved.
Audit Log
Complete record of every bill processed: outcome, timestamp, whether Claude and Gemini agreed.
providers.json
Registry of known utility providers in the GitHub repo. When a new provider appears, the system alerts before processing. After you verify and add it to the registry, it processes automatically going forward.
bills.json
Data file written by Apps Script after every run. This dashboard reads it. Contains summary counts, recent bill log, Review Queue items, and system links.
