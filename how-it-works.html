Data Flow
1. Bill arrives in Google Drive
A utility bill PDF is dropped into the designated Drive folder. All bill types go into the same folder. The system identifies the type and provider automatically.
2. Apps Script detects the new file
A time-based trigger runs every 15 minutes, comparing files against a log of already-processed IDs. New files are queued for processing.
3. Provider check against known registry
The detected provider is checked against providers.json. If new, a "New Provider Detected" alert fires for manual review before any data is written.
4. Claude API extracts all data fields
The full PDF is sent to Claude. It returns structured JSON: bill type, property ID, dates, usage, cost, estimation flag, demand, and demand cost.
5. Gemini API independently validates
The same PDF goes to Gemini with identical instructions but without Claude's output. Gemini reaches its own conclusions independently.
6. Outputs are compared and validated
Numeric values must agree within 2%. Dates must match exactly. Sanity checks run on all required fields regardless of agreement.
7a. Agreement → written to Sheet
If both APIs agree and all checks pass, data is written to the correct tab with a sequential row number and the Audit Log is updated.
7b. Disagreement → Review Queue + email alert
If the APIs disagree or sanity checks fail, the bill goes to Review Queue and an email alert fires with the row number, filename, conflicting fields, and both values.
8. Dashboard updated
After each run, bills.json is written to this GitHub repo. The dashboard reads it and refreshes every 60 seconds.
New Provider Detection

The system maintains a registry of known providers in data/providers.json. When a bill arrives from an unrecognized provider:

  1. A New Provider Detected email fires with the full Claude extraction.
  2. Verify the extracted values manually against the actual bill.
  3. If correct, add the provider name to the known array in data/providers.json.
  4. On the next run, the system trusts that provider automatically.
You never need to write new parsing rules
The AI handles layout variation naturally. The registry just tracks which providers have been verified. After the first confirmation, all future bills from that provider process automatically.

When a Bill Is Flagged
You will receive an email alert
Subject: [Bill Parser] Review Required - Row #X - Electricity
The email includes the row number, filename, which fields disagreed, and both extracted values side by side.
  1. Open the email. Note the row number and which fields are in conflict.
  2. Open the original bill using the Drive link in the email.
  3. Look up the correct value directly from the bill.
  4. Open the Google Sheet → Review Queue tab. Find the row by number.
  5. Enter correct values into the appropriate utility tab. Mark the row as RESOLVED.
The system never writes wrong data
A bill in Review Queue is always better than a wrong number in the sheet.
Security

All credentials are stored in Google Apps Script Script Properties. No keys are written in code or committed to GitHub.

Keys stored in Script Properties
  • CLAUDE_API_KEYAnthropic API key for primary extraction
  • GEMINI_API_KEYGoogle Gemini API key for validation
  • GITHUB_TOKENFine-grained PAT, repo contents:write only
  • GITHUB_REPOkeyona-rerev/TESTER_Resource-Bill-Parser-Github
  • SHEET_IDGoogle Sheet ID from the URL
  • DRIVE_FOLDER_IDDrive folder ID where bills are dropped
  • ALERT_EMAILEmail address for all system alerts
Glossary
kWh (kilowatt-hour)
Unit of electricity consumption. Usage is the unadjusted, pre-loss-factor figure.
m³ (cubic metres)
Unit of volume for natural gas and water bills.
Demand (kW)
Peak power draw during the billing period. Electricity only. Uses the unadjusted Demand kW figure.
Demand Cost
Distribution + Transmission Network + Transmission Connection − Transformer Allowance (if any).
Ontario Electricity Rebate
Provincial credit shown as "CR." Subtracted from the electricity cost.
This Cost You (Enbridge)
Natural gas cost on page 1, already including HST. Taken as-is.
Estimation (Yes/No)
Whether the meter reading was estimated (Yes) or an actual read (No).
Review Queue
Sheet tab for bills where the AIs disagreed or sanity checks failed. Nothing writes to utility tabs until manually resolved.
Audit Log
Complete record of every bill processed: outcome, timestamp, whether Claude and Gemini agreed.
providers.json
Registry of known providers. Add a provider name here after verifying its first bill manually.
bills.json
Written after every run. Contains summary counts, recent log, Review Queue, and system links.
